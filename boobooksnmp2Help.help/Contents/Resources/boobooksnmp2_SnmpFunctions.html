SNMP Functions

  • Get
  • Walk
  • Get Next
  • Set
  • Get Bulk
  • Cancel All Requests
Basic set of SNMP functions is supported. Requests are threaded for simultaneous execution.
All output is displayed in Query Results View.

Get

Executes a SNMP Get request for data in Agent View and Object view.

Walk

Executes a SNMP Walk request for data in Agent View and Object view. SNMP walk function executes successive snmp get next requests and these requests are sequential, that is one thread is created for snmp walk function that in turn executes synchronous(one at a time) snmp get next request.

Get Next

Executes a SNMP Get Next request for data in Agent View and Object view.

Set

Executes SNMP Set request for date in Agenet View and Object View.

Get Bulk

SNMP Bulk implementation is done as to enable ability to query multiple Agents for multiple Objects. Selecting this action will trigger collection of all Agents and Objects selected in Bookmarks View and executing (queuing) of SNMP GET request for each Agent for each Object. All requests are threaded and are executed as fast as the OS can process them.
Selecting a Folder in Bookmark View means you are selecting all Agents and Objects in that Folder.

Cancel All Requests

SNMP requests are threaded so each request is queued and executed in background. This has flexibility to schedule multiple requests and not have to wait until first task finished executing. Results are populated in current Query Results View in order tasks are finished executing. As a consequence to cancel a request means stopping all requests, and suppressing any further output.
