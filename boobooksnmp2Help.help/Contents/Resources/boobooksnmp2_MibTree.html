MIB Object Tree

MIB tree displays Objects read in by libnetsnmp. MIB files read in are set in Preferences.
Selecting an Object in the MIB tree will populate that object information in the Object section.

Loading MIBs

MIBs are loaded in the order listed in Preferences. Per libnetsnmp behavior, last file read in that defines a SNMP Modue replaces any previous Module definition loaded. Use the Object "File" field to see which Module and MIB file OID definition comes from.

If present, MIB file load errors are reported in Log View. Level of output depends on Log Level set in Preferences. Errors are repoted in order MIB files are read in and is a good way to troubleshoot MIB files, as well as load order.
Having an error reported that a module could not be found does not necesseraly mean that Module was not loaded. Applciation (libnetsnmp) attempts to adopt Modules at the end of reading in all MB files, ones it could not at MIB file read time.

Search

Search field above the MIB Object tree is used to search for an Object by string of characters. Search function uses libnetsnmp "find_oid_in_tree()" function which searches for "^Begins with".
Selecting an object from the list will highlight it in MIB Object Tree.
Search results returned are limited to 512 entries.
